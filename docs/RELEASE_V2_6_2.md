# FailLens v2.6.2 Value and Objection Pack

Status: public docs release

This release adds public value-prep documents for the main buyer scenarios and objections.

## Added

- WHY_XPAY_FIRST
- WHY_NOT_TAIL_GREP
- AGENT_BUILDER_VALUE
- SOLO_DEVELOPER_QUICKSTART
- CI_VALUE
- SECURITY_POSTURE

## Purpose

v2.6.2 prepares FailLens for outreach and adoption proof.

The goal is to answer the common questions before users ask them:

- Why pay through xpay?
- Why not just use tail or grep?
- Why should agents call FailLens?
- How does FailLens save money?
- What does CI gain?
- What is the security posture?
- What does FailLens not claim to be?

## Positioning

FailLens remains narrow:

    failed command -> redacted bounded evidence -> agent-safe next step

## Business rule

xpay remains the default paid path.

Subscriptions are not the immediate public buying path.

Subscriptions may come later for predictable team workflows, usage visibility, support, and CI automation.
